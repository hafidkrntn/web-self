SITO

System Tracking Oil

Software Engineer — Product Planning, UI/UX, Frontend, Backend, Database, Deployment

Problem

Most riders estimate oil replacement manually, which often results in replacing oil too early or too late. There is no simple application focused on personalized maintenance reminders.

Solution

Built as a mobile-first Progressive Web App using Astro for the frontend and Go for the backend with PostgreSQL as the database. Designed with a scalable architecture to support future features such as push notifications, maintenance history, and analytics.

Highlights
  • Mobile-first PWA with Astro frontend and Go backend
  • PostgreSQL-backed architecture designed for scalability
  • Engineered to support push notifications, maintenance history, and analytics
Stack
AstroGoPostgreSQL
Outcome

Focused on balancing simplicity with scalability. The architecture is prepared for future growth while remaining easy to use.

Approach

I build software
that solves problems
without creating new ones.

I don't build software to use interesting technology. I build it to solve real problems. An app that helps riders maintain their motorcycles. A platform that replaces manual processes with integrated workflows. A website that communicates clearly and loads fast.

Every decision is a tradeoff. The right database, the right level of abstraction, the right amount of code. I prefer code that's predictable — easy to understand, easy to change, and easy to trust. Code that's boring ships faster, breaks less, and lasts longer.

02 Keep it simple.
